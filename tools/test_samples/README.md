# Test samples

Three representative source files for testing the content pipeline. Each demonstrates a different real-world starting point an agency might have.

| File | Type | What it represents |
|------|------|---------------------|
| `sample_pub1075_excerpt.pdf` | PDF | A regulatory publication-style document with numbered sections (1.0, 1.1, 2.0...). Mimics the structure of IRS Pub 1075. |
| `sample_training.docx` | DOCX | An existing training narrative as a Word doc with explicit heading styles, body text, and bulleted lists. |
| `sample_training.pptx` | PPTX | An existing slide-deck training with title slides, bulleted content, and speaker notes. |

The corresponding `extracted_*.json` files are the output of `extract_content.py` on each source. The `draft_*.md` files are the output of `draft_training.py` on each JSON.

## Reproduce the test outputs

```bash
cd tools/test_samples

# Extract from each source
python3 ../extract_content.py sample_pub1075_excerpt.pdf -o extracted_pdf.json
python3 ../extract_content.py sample_training.docx -o extracted_docx.json
python3 ../extract_content.py sample_training.pptx -o extracted_pptx.json

# Generate drafts from each
python3 ../draft_training.py extracted_pdf.json -o draft_pdf.md
python3 ../draft_training.py extracted_docx.json -o draft_docx.md
python3 ../draft_training.py extracted_pptx.json -o draft_pptx.md
```

## What to look at

- **`draft_pptx.md`** is the most representative of the typical workflow — most agencies have an existing PowerPoint training that needs to become structured content. Read this draft first.
- **`extracted_pptx.json`** shows the structure preserved from the source — note that speaker notes are captured (often where the real content lives in compliance trainings).
- **`draft_pdf.md`** demonstrates extraction from a regulatory publication. Note how the legal references (IRC §§ 6103, 7213, 7213A, 7431) are auto-detected at the bottom of the draft.

These samples are not real client/agency materials — they were generated by Python scripts using `reportlab`, `python-docx`, and `python-pptx` to test the pipeline. Real source documents will produce richer extracted content.
